# FreeCAD MCP Server Plan for Parametric Parts & Assemblies

## 1) Goals
- Provide an MCP server that lets AI agents create, edit, analyze, and export **FreeCAD** models.
- Make all models **fully parametric** and **readable** by driving dimensions from spreadsheets.
- Support both **single parts** and **multi-part assemblies** with traceable design intent.
- Add “intelligent design consideration” through rule checks, optimization suggestions, and manufacturability feedback.

## 2) Core Design Principles
1. **Parametric-first**: no hard-coded sketch dimensions; all values come from named spreadsheet cells.
2. **Readable parameters**: engineering-style names (e.g., `plate_length_mm`, `bolt_hole_dia_mm`).
3. **Single source of truth**: each part has one parameter table; assemblies have one top-level parameter table.
4. **Deterministic generation**: same parameter set always generates the same geometry.
5. **Non-destructive updates**: edits update parameters/features, not manual direct-geometry hacks.

## 3) Proposed System Architecture

### 3.1 MCP Server Layers
- **Transport/API Layer**
  - MCP tool registration, request validation, auth/session handling.
- **Design Orchestration Layer**
  - Interprets user intent into structured CAD operations.
  - Maintains operation graph (create/update/delete features).
- **FreeCAD Adapter Layer**
  - Wraps FreeCAD Python APIs (PartDesign, Sketcher, Spreadsheet, Assembly).
  - Handles document lifecycle, recompute, save/load, and export.
- **Validation & Intelligence Layer**
  - Constraint checks (over/under constrained sketches).
  - Rules: minimum wall thickness, clearance, fillet/chamfer limits, hole edge distances.
  - Suggestions: material-aware dimension hints, tolerance warnings.
- **Persistence Layer**
  - Local workspace store for `.FCStd`, templates, and metadata.
  - Optional git-backed versioning of param snapshots.

### 3.2 Domain Model (Canonical Objects)
- **Project**
  - multiple `PartModels`, optional `AssemblyModel`, global metadata.
- **PartModel**
  - `Spreadsheet` (named params), `Sketches`, `Features`, `Bodies`, derived properties.
- **AssemblyModel**
  - references to part files/configurations, mating constraints, BOM metadata.
- **ParameterSet**
  - typed values (length, angle, count, string, bool), units, bounds, descriptions.
- **DesignRuleSet**
  - rule definitions and severity levels (info/warn/error).

## 4) Parameter Strategy (Spreadsheet-Centric)

### 4.1 Naming Convention
- Prefix by scope:
  - `g_` for global assembly-level parameters.
  - `p_` for part-level parameters.
  - `m_` for manufacturing/tolerance parameters.
- Example names:
  - `p_base_len_mm`, `p_base_w_mm`, `p_thickness_mm`, `m_clearance_fit_mm`.

### 4.2 Spreadsheet Layout Standard
Columns:
1. `Name`
2. `Expression`
3. `Unit`
4. `Value`
5. `Min`
6. `Max`
7. `Description`
8. `Category`

Rules:
- Every driven dimension references `Spreadsheet.<Name>`.
- No unnamed dimensions in sketches.
- Each param must include `Description` and bounds.

### 4.3 Template Parameter Packs
- **Fastener pack**: thread size, head clearance, nut trap dimensions.
- **Sheet metal pack**: thickness, bend allowance, K-factor.
- **Enclosure pack**: wall thickness, standoff offsets, lid overlap.

## 5) Assembly Approach
- Prefer Assembly workbench compatible with current FreeCAD stable APIs.
- Assembly-level spreadsheet defines interfaces:
  - mount pattern pitch, datum offsets, shaft center heights, envelope limits.
- Parts consume interface parameters via linked expressions.
- Mates/constraints generated by named reference geometry:
  - datum planes, coordinate systems, and mating axes.

## 6) MCP Tooling Surface (Initial)

### 6.1 Project/Document Tools
- `project.create`
- `project.open`
- `project.save`
- `project.export` (STEP/STL/DXF)

### 6.2 Parameter Tools
- `param.list`
- `param.set`
- `param.batch_set`
- `param.validate` (bounds + unit checks)

### 6.3 Part Modeling Tools
- `part.create_from_template`
- `sketch.create`
- `sketch.add_profile`
- `feature.pad`
- `feature.pocket`
- `feature.fillet`
- `feature.chamfer`

### 6.4 Assembly Tools
- `assembly.create`
- `assembly.add_part`
- `assembly.mate`
- `assembly.explode_view`
- `assembly.bom`

### 6.5 Intelligence Tools
- `design.check_rules`
- `design.optimize_parameter` (single-objective first)
- `design.suggest_improvements`

## 7) Intelligent Design Consideration (Practical AI)
- **Intent capture**: parse requirements into measurable constraints (strength, mass, envelope).
- **Constraint-aware edits**: reject geometry changes that violate rule thresholds.
- **Design scoring**: weighted score from manufacturability, assembly simplicity, and mass.
- **What-if sweeps**: run param sweeps and rank candidates.
- **Explainability**: every suggestion includes “why” and impacted parameters/features.

## 8) Safety, Robustness, and Traceability
- Transaction model per MCP command:
  1. snapshot,
  2. apply,
  3. recompute,
  4. validate,
  5. commit or rollback.
- Structured logs for each operation with:
  - tool name,
  - touched parameters/features,
  - rule check results.
- Crash-safe autosave every successful transaction.

## 9) Suggested Implementation Phases

### Phase 1 — Foundation (MVP)
- Basic MCP server shell.
- FreeCAD document open/save.
- Spreadsheet parameter CRUD.
- Simple part generation from template sketch + pad.

### Phase 2 — Parametric Part Authoring
- Feature library (pocket, fillet, chamfer).
- Naming/bounds validation.
- Export pipeline (STEP/STL).

### Phase 3 — Assembly + BOM
- Add assembly creation and mates.
- Linked parameters across parts.
- BOM extraction and metadata.

### Phase 4 — Intelligent Checks
- Rule engine with configurable thresholds.
- Suggestion engine and parameter sweep utility.
- Design score and recommendation explanations.

### Phase 5 — Production Hardening
- Regression test suite for deterministic geometry.
- Performance profiling on large assemblies.
- Versioned templates and migration strategy.

## 10) Test Strategy
- **Unit tests**: parameter parsing, unit conversion, bound checks.
- **Integration tests**: end-to-end part build and assembly mate workflows.
- **Golden-model tests**: compare generated volumes/masses/reference dimensions.
- **Fuzz tests**: random but bounded parameter sets.
- **Rule tests**: verify warnings/errors for known bad configurations.

## 11) Deliverables
- MCP server with documented tool schema.
- Parametric templates for common mechanical components.
- Example project: multi-part assembly with top-level spreadsheet and BOM.
- Operator guide with naming conventions and best practices.

## 12) Definition of Done
- Every geometry-driving number comes from a named spreadsheet parameter.
- Assemblies rebuild after top-level parameter edits without manual fixups.
- Rule checker catches critical manufacturability issues.
- Exports succeed for validated configurations.
- Logs explain every automated design decision.
